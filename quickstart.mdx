---
title: "Get started with NEXUS"
description: "Learn how to install NEXUS, upload your first research paper, and ask your first RAG-powered question with citations in under 5 minutes."
---

This guide walks you through cloning the repository, configuring the environment, starting the full stack with Docker Compose, and asking your first question against a research paper.

<Note>
  Before you begin, make sure you have the following available:

  - **Docker & Docker Compose** installed and running
  - A **running Ollama instance** with at least one model pulled, for example:
    ```bash
    ollama pull llama3.2:3b
    ```
  - **Qdrant** either already running locally or started via Docker (step 3 handles this automatically when using Docker Compose)
</Note>

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/gitInfinity/nexus.git && cd nexus
    ```
  </Step>

  <Step title="Create the environment file">
    Create a file at `backend/.env` with the following content. The defaults below match the values in `backend/core/config.py` and will work out of the box for a local Docker Compose deployment.

    ```env
    OLLAMA_URL=http://localhost:11434
    OLLAMA_MODEL=llama3.2:3b
    QDRANT_URL=http://qdrant:6333
    QDRANT_COLLECTION=nexus_papers
    DATABASE_URL=sqlite+aiosqlite:///./nexus.db
    EMBEDDING_MODEL=BAAI/bge-m3
    MAX_CHUNK_SIZE=512
    CHUNK_OVERLAP=64
    TOP_K_RETRIEVAL=8
    UPLOAD_DIR=./uploads
    CORS_ORIGINS=http://localhost:5173,http://localhost:3000
    ```

    The backend uses `pydantic-settings`, so each key must match the field name exactly (case-insensitive).
  </Step>

  <Step title="Start the stack">
    ```bash
    docker compose up --build
    ```

    Docker Compose starts three services:

    | Service | Port | Description |
    |---|---|---|
    | `qdrant` | 6333 | Vector database with persistent storage |
    | `backend` | 8000 | FastAPI server with hot-reload enabled |
    | `frontend` | 5173 | Vite development server |

    Wait until you see `NEXUS backend ready.` in the logs before proceeding.
  </Step>

  <Step title="Upload a paper">
    Open [http://localhost:5173](http://localhost:5173) in your browser.

    Click the **⊕ Ingest** button in the top bar, then drag and drop one or more PDF files into the upload dialog. Each paper moves through three states visible in the papers list:

    1. **queued** — file received, waiting for the embedding pipeline
    2. **processing** — chunks are being embedded and sent to Qdrant
    3. **indexed** — paper is ready to query

    Indexing time depends on document length and your hardware. A typical 10-page PDF takes 10–30 seconds.
  </Step>

  <Step title="Create a research session">
    Click **⊕ New Session** in the left sidebar. Give the session a name, then select the papers you want to include from the paper library. Only papers marked **indexed** can be added to a session.
  </Step>

  <Step title="Ask your first question">
    Type a question in the chat input and press **Enter**. NEXUS retrieves the top-8 most relevant chunks from the session's papers and streams a grounded answer with inline citations.

    Example questions to try:

    ```text
    What are the key findings of this paper?
    What research gaps does the author identify?
    Summarize the methodology used in this study.
    How do the results compare across the papers in this session?
    ```

    Each response includes source references pointing back to the specific paper and chunk that informed the answer.
  </Step>
</Steps>

<Tip>
  You can verify that the backend is healthy at any time with:

  ```bash
  curl http://localhost:8000/health
  ```

  A successful response looks like `{"status":"ok","model":"llama3.2:3b"}`.
</Tip>
