NEXUS exposes a REST API built with FastAPI (v1.0.0). All endpoints are prefixed with `/api`. The API has no authentication by default — add a reverse proxy with auth for production deployments.

## Connection details

| | |
|---|---|
| **Base URL** | `http://localhost:8000` |
| **Interactive docs** | `http://localhost:8000/docs` (Swagger UI) |
| **Health check** | `GET /health` → `{"status": "ok", "model": "<model name>"}` |

```bash
curl http://localhost:8000/health
# {"status": "ok", "model": "llama3.2:3b"}
```

## Request and response format

All request bodies are JSON (`Content-Type: application/json`), except file uploads which use `multipart/form-data`. All responses are JSON except export endpoints, which return binary file downloads.

Error responses follow FastAPI's standard format:

```json
{"detail": "error message"}
```

## Common HTTP status codes

| Code | Meaning |
|---|---|
| `200 OK` | Request succeeded |
| `400 Bad Request` | Invalid parameters (e.g. unsupported export format, empty session) |
| `404 Not Found` | Resource does not exist |
| `422 Unprocessable Entity` | Request body failed schema validation |
